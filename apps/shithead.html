The classic card game, online

Shithead is a shedding-type card game that's been played at kitchen tables, campsites, and hostels for decades. The goal is simple: get rid of all your cards before everyone else. The last player holding cards is the Shithead.

This version brings it online with real-time multiplayer, bot opponents, in-game chat, and enough polish to feel like a proper game — not a prototype. No account required. Create a room, share the link, and play.

Features

  • Real-time multiplayer — up to 4 players via WebSocket, synchronised game state
  • Bot opponents — fill empty seats with bots and start immediately
  • Lobby chat — chat before and during the game with reaction sounds
  • Drag-to-sort hand — rearrange your cards before locking them face-up
  • Turn and game timers — keeps the game moving, auto-plays on timeout
  • Quick-play mechanic — multiple identical cards can be played at once
  • Ready badges — see when each player is ready to start
  • Idle poop animation — the Shithead earns a special animation while waiting their turn
  • Mute / surrender controls — step away gracefully
  • 24h game expiry — abandoned games clean up automatically

The goal

Get rid of every card in your hand, then your face-up cards, then your face-down cards. The last player with cards is the Shithead.

Setup

Each player is dealt 9 cards: 3 face-down (locked, don't look), 3 face-up on top of the face-down pile (you choose which 3 before the game starts), and 3 in hand. Before the game begins, you can swap hand cards for face-up cards to build a stronger final layer.

Playing

  • Play a card equal to or higher than the top card of the discard pile.
  • Play multiple cards of the same rank at once.
  • If you can't play, pick up the entire discard pile.
  • Special cards: 2 resets the pile (anything can be played next), 10 burns the pile (it's discarded, play again), 7 reverses the rule (must play equal or lower).

End game

Once your hand is empty, play from your face-up cards. Once those are gone, flip and play face-down cards blind — no peeking. If a face-down card can't be played, you pick up the pile. The first player to clear all three layers wins. The last to clear theirs is the Shithead.

Why this game

Shithead is one of those games that almost everyone has played at some point, but nobody could ever find online in a form that felt right. Most versions are buried in Flash-era sites or require account creation. This one is just a link.

Building with Claude Code

The game engine was the core challenge. Shithead has a deceptively complex rule set — the three-phase hand structure (hand → face-up → face-down blind), the pile-reset and pile-burn special cards, the 7-reverse rule, and the edge cases around playing multiple identical cards all had to be implemented correctly before anything else could work.

Claude Code built the full game engine, real-time Socket.IO synchronisation, and the React frontend in one extended session. Bot logic came next — bots needed to make legal moves without breaking the game state, with a preference for burning the pile when possible.

Polish came last: drag-and-drop hand ordering, the idle poop animation, fart sound effects, turn timers with auto-play, and lobby chat with sound reactions. Each feature was described conversationally and implemented incrementally.

"The face-down phase is where the game gets truly chaotic. You flip blind, hope for the best, and sometimes pick up the entire pile. Getting that feeling right required careful attention to timing and animation."

The quick-play mechanic

One of the trickier implementation details: if you have multiple cards of the same rank, you can play them all at once. If those cards include a 10, the pile burns immediately. If combined with a 7, the reverse rule applies to the entire play. All these interactions had to be resolved in the right order, server-side, with the result broadcast to all clients atomically.

Architecture

The game is split into a React frontend hosted on Firebase and a Node.js + Express backend running on Google Cloud Run. All game state lives on the server — the client is a pure renderer. Socket.IO handles real-time communication between players and bots.

Game rooms expire after 24 hours of inactivity. Abandoned sockets are cleaned up on disconnect. The server uses an in-memory store for active games (no database required at this scale).
