OtočSvět

A map tile puzzle — scrambled tiles of real city maps, rotated back into place. The faster you solve it, the higher the score.

Play Beta
🗓️ 22 Apr 2026 🤖 Built with Claude Code 🌍 React · Fastify · PostgreSQL · MapTiler

What is OtočSvět?

OtočSvět (Czech for "Rotate the World") is a puzzle game built around real geography. Each round, you're shown a grid of map tiles — satellite-quality images of an actual city, cut into pieces and scrambled with random rotations. Your job is to click each tile back into its correct orientation.

It sounds simple. In practice, distinguishing a rotated road intersection from a rotated riverbank at speed is genuinely tricky — and satisfying when it clicks.

Game modes

  • Solo — 5 puzzles in a row, your choice of grid size: 3×3, 4×4, or 5×5. Score is based on correctness and speed.
  • Daily challenge — a fixed set of 5 puzzles, same for everyone on that day. Compare scores on the leaderboard.

Scoring

Each correct solution earns up to 1,000 points. The time bonus decays from the moment a round starts — the faster you confirm, the more you keep. Wrong attempts cost 200 points each. Skip a round for 0 points and move on. Gold medal starts at 3,500 total points across 5 rounds.

The cities

Puzzles are generated from city maps across the world using the MapTiler Cloud API — the same high-quality tiles used in many professional mapping tools. Every puzzle is reviewed and rated for difficulty by a human before it goes live. Unsolvable or ambiguous puzzles (too much water, featureless rooftops) are filtered out.

Still in beta

The game is live and playable. The puzzle library is growing. Multiplayer arena mode — where players race through the same puzzle simultaneously — is planned as phase two.

The idea

The starting point was a whitepaper — not a technical spec, but a one-page game design document describing what the game should feel like: fast, geographic, satisfying. The constraint of "real maps" as the puzzle surface felt unique. Every puzzle is inherently different because every city looks different.

Building with Claude Code

The backend puzzle generator was the first challenge. MapTiler's static API returns image tiles at specific coordinates and zoom levels — the generator slices those into a grid, stores the tiles, and records their correct rotations. Claude Code wrote the CLI tool, the Fastify API, and the Prisma schema in a single session.

The frontend game loop was built next: loading a session, showing tiles, handling rotations, computing scores on the server, advancing rounds. The tile rotation animation alone went through three rewrites before the CSS transition approach was abandoned in favour of a per-tile animation component that didn't fight React's rendering cycle.

The admin panel was the last piece — a password-protected review interface where puzzles can be approved, rated for difficulty (1–5), and assigned to daily challenges. Rate limiting was added after the first load test revealed the review endpoint had no protection against rapid-fire submissions.

The hardest part

Session resume. If a player closes the tab mid-game, the session should survive. If they refresh, the timer should account for elapsed time. Getting the client and server to agree on game state across disconnects required careful design — the server is the authority, the client reconstructs from the session payload on resume.

"The rotation animation going from CSS transitions to a React component was the right call. It stopped fighting the state machine."

Architecture

The project is a pnpm monorepo with three packages: backend (Fastify API + Prisma + puzzle generator CLI), frontend (React + Vite + Tailwind), and shared (TypeScript types shared across both). The frontend is deployed to Vercel. The backend runs in a Docker container.

Puzzle generation

Puzzles are generated offline by a CLI tool: pnpm generate -- --gridSize 3 --count 10. It calls the MapTiler Static API for a random city coordinate, downloads the image, slices it into an N×N grid of tiles, stores them as WebP files, and records each tile's correct rotation (always 0 — the initial scramble is applied at session start). Generated puzzles start in draft status and must be reviewed and activated before players see them.

Scoring

Score calculation runs server-side on submission. Time bonus starts at 500 points and decays at 5 points per second from when the round began (server timestamp). Wrong attempts subtract 200 from the final round score. The server validates the submitted rotations against the stored correct values and returns the earned score immediately.
