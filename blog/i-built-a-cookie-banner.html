I Needed a Cookie Banner.
So I Built One — Free for Everyone.

Refused to pay another monthly fee for a cookie consent tool. So I described what I wanted to Claude Code, and ConsentKit was born — open source, fully featured, and yours to use at zero cost.

When I set up Resonant Labs, I needed a proper cookie consent banner. GDPR is real, Google Consent Mode v2 is real, and if you're running analytics you need to handle it properly. So I did what most people do: I started looking at solutions.

The options were fine. CookieYes, Cookiebot, Termly — all solid, all professional. Also all behind a subscription. Monthly fees for a widget. Something that sits on your site and shows a popup. Something that, fundamentally, just needs to store a few values in localStorage and maybe send an event to Google.

I decided not to pay.

The problem with most cookie tools

It's not that the tools are bad. It's that the pricing model doesn't make sense once you understand what cookie consent actually requires technically. At its core, a GDPR-compliant banner needs to:

  • Show a banner and collect consent by category
  • Store the user's preferences
  • Block third-party scripts until consent is granted
  • Support Google Consent Mode v2 signals
  • Let users change their mind later

That's it. There's no reason this should cost money every month. There's no reason a small personal site or a startup should be paying the same rate as a large e-commerce platform just to put a popup on their homepage.

"Monthly fees for a widget that stores a preference in localStorage felt absurd. So I described what I wanted to Claude Code and built it myself."

What I built

ConsentKit is an open-source cookie consent widget built entirely through AI conversation. It's a single JavaScript file — 20KB raw, under 6KB gzipped — that you drop into any website. No server required. No account. No subscription.

It supports four consent categories out of the box: Necessary, Analytics, Marketing, and Functional. The banner appearance is fully configurable — colours, position, copy, everything — through a simple JSON config. And it integrates natively with Google Consent Mode v2, automatically sending gtag('consent', 'update', ...) when visitors make their choice.

Some things I wanted that most paid tools still get wrong:

  • Shadow DOM isolation — the banner's styles never bleed into your site
  • Scroll lock — visitors can't browse past the banner without making a choice
  • Reopener button — a persistent icon so users can always change their preferences
  • Static site friendly — works with no backend, config lives inline in the page
  • Fully on-brand — I wanted it to look like part of the site, not a generic popup

The build process

I described the full widget to Claude Code in a single session: the architecture, the config schema, the Shadow DOM approach, the Google Consent Mode integration. We worked through it layer by layer — the banner UI, the category toggles, the localStorage persistence, the reopener button.

What came out the other side was clean TypeScript with an esbuild pipeline. I shaped every decision through conversation — which categories to include, what the copy should say, how the animation should feel, where the reopener button should sit. Claude Code handled the implementation. I handled the vision.

That's the dynamic that makes this work. It's not that AI does it all on its own. I still think through every concept, guide each step, and make every meaningful decision. But instead of writing code, I do it by talking. My thinking, my instincts, my product sense — implemented through conversation rather than syntax.

Why free and open source

The logic is the same as everything else I build here. The code was generated by AI. I contributed the thinking, the product decisions, the time spent iterating. But the actual implementation — the TypeScript, the CSS, the build config — that came from Claude Code. Charging for it didn't feel right.

Beyond the ethics, there's a practical argument: the world would be better if more small sites, indie projects, and solo developers had access to a proper consent tool without paying a monthly fee. Privacy compliance shouldn't be a privilege for those who can afford the subscription.

So here it is. MIT licensed. Free forever.

This is one of the best AI use cases

I keep coming back to this pattern: don't outsource, don't subscribe, don't pay for something a good AI conversation can build from scratch in an afternoon. ConsentKit cost me a few dollars of API usage and a few hours of focused thinking. The result is something I'd have previously paid for every month, indefinitely.

The savings compound immediately. No recurring cost. Full control. Exactly the features I want, none I don't. I can extend it whenever I need to. And now anyone else can use it too, for free, for any project.

If you need cookie consent for your site, use it. If you want to extend it or improve it, fork it. If you'd like something similar built for a different problem you're paying to solve — send me the idea.
